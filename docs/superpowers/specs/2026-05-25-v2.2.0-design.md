# WutheringWavesSteamHelper v2.2.0 设计文档

- **日期**: 2026-05-25
- **分支**: `v2.2.0`
- **基线版本**: v2.1.0
- **目标版本**: v2.2.0

## 1. 需求概述

v2.2.0 引入三项功能改动，整体属于增量重构 + 新增页面，不影响 v2.1.0 已稳定的鸣潮主流程：

1. **全局化 Steam 配置**：将"Steam 安装路径 / Steam 库路径 / SteamID"从鸣潮页搬迁到设置页，作为应用级全局配置，供鸣潮页与新增的"自定义 Manifest"页共享。
2. **新增侧边栏入口**：在 `NavigationView.MenuItems` 增加"自定义 Manifest"项。
3. **新增自定义 Manifest 页面**：与鸣潮页布局相近，但允许用户自定义 AppID、DepotID、BuildID、Manifest、显示名、安装目录名、可执行文件路径、ACF 语言代码；保留"生成 ACF 配置文件""复制启动命令""运行日志"等核心功能。

## 2. 设计决策

### 2.1 持久化策略（A 实现 + B 预留）

本期采用 **方案 A（单组自定义配置）**，但数据模型同时预留 **方案 B（多预设列表）** 的扩展位，避免未来升级时迁移历史数据。

```csharp
// AppSettings 新增字段
public CustomManifestPreset CurrentCustomManifest { get; set; } = new();
public List<CustomManifestPreset> CustomManifestPresets { get; set; } = new(); // v2.3 启用
```

v2.2.0 阶段 UI 仅读写 `CurrentCustomManifest`；`CustomManifestPresets` 字段被序列化为空数组，但不暴露任何管理 UI。v2.3 升级时只需新增"预设选择 ComboBox"和"另存为预设"对话框，无需迁移 settings.json。

### 2.2 保存语义

自定义页采用**显式保存**（"保存当前配置"按钮），而非鸣潮页风格的"控件变更即写盘"。原因：

- 自定义页字段更多（9 个字段），自动写盘会在用户调试 AppID/DepotID 时产生大量无意义磁盘写入。
- "保存"按钮的语义与未来 B 阶段"另存为预设"自然衔接。

### 2.3 鸣潮页搬迁后是否保留入口提示

不保留。鸣潮页删除"Steam 安装路径""库路径""SteamID"三张卡片后，直接由顶部到"国服游戏来源"。理由：用户已在设置页配置过一次，鸣潮页不应再重复提示。

### 2.4 自定义页是否需要"打开启动器"

不需要。该功能为鸣潮国服专属（启动 `launcher.exe` 后由其拉起客户端），与"自定义任意 Steam 游戏"语义无关。

## 3. 架构变更

### 3.1 数据模型 (`Models/AppSettings.cs`)

```csharp
public class AppSettings
{
    // ===== 全局通用 Steam 配置（v2.2.0 后由设置页管理）=====
    public string SteamInstallPath { get; set; } = "";
    public string SteamLibraryPath { get; set; } = "";
    public string SteamId { get; set; } = "";

    // ===== 鸣潮专用（保留不动）=====
    public string BuildId { get; set; } = "";
    public string Manifest { get; set; } = "";
    public string CnGameSource { get; set; } = "official";

    // ===== 自定义 Manifest（v2.2.0 新增）=====
    public CustomManifestPreset CurrentCustomManifest { get; set; } = new();
    public List<CustomManifestPreset> CustomManifestPresets { get; set; } = new(); // 预留 v2.3

    // ===== 应用设置（保留不动）=====
    public bool DeveloperMode { get; set; }
    public bool DebugMode { get; set; }
    public bool BetaChannel { get; set; }
    public string Language { get; set; } = "zh-CN";
}

public class CustomManifestPreset
{
    public string Name { get; set; } = "默认";
    public string AppId { get; set; } = "";
    public string DepotId { get; set; } = "";
    public string BuildId { get; set; } = "";
    public string Manifest { get; set; } = "";
    public string GameDisplayName { get; set; } = "";
    public string InstallDir { get; set; } = "";
    public string ClientExePath { get; set; } = "";
    public string Language { get; set; } = "schinese";
}
```

### 3.2 服务层 (`Services/SteamService.cs`)

```csharp
public sealed record AcfParameters(
    string AppId,
    string DepotId,
    string LauncherPath,
    string DisplayName,
    string InstallDir,
    string BuildId,
    string LastOwner,
    string Manifest,
    string Language = "schinese");

public class SteamService
{
    // 新签名：参数化 ACF 生成（鸣潮页与自定义页共用）
    public string GenerateAcfContent(AcfParameters p);

    // 鸣潮页便捷包装（保持调用点简洁）
    public string GenerateAcfForWutheringWaves(
        string launcherPath, string buildId, string steamId, string manifest);

    // 新签名：参数化 SteamDB 查询
    public Task<(string buildId, string manifest)?> FetchSteamDbInfoAsync(
        string appId, string depotId);

    // 自定义页用：直接基于用户给定的 exe 路径生成启动命令
    public string GenerateLaunchCommandFromExe(string exePath)
        => $"\"{exePath}\" %command%";
}
```

`SharedDepots` 段（`228989 -> 228980`，Steamworks Common Redistributables）按通用处理保留不变；遇到不兼容游戏可在 v2.3 引入"高级选项"暴露此字段。

### 3.3 UI 层

#### 设置页 `Views/Pages/SettingsPage.xaml`

在"开发者选项"卡片之前插入新卡片 **"Steam 全局配置"**，包含三个子区块：

- Steam 安装路径（TextBox + 浏览 + 自动检测）
- Steam 游戏库路径（TextBox + 浏览 + 自动检测）
- Steam ID（TextBox + 帮助提示）

控件 `Name` 沿用：`txtSteamPath / txtLibraryPath / txtSteamId`；事件处理器从 `WutheringWavesPage` 平移至 `SettingsPage`。

#### 鸣潮页 `Views/Pages/WutheringWavesPage.xaml`

删除以下三张 `<Border>` 卡片：
- "Steam 安装路径"
- "Steam 游戏库路径（SteamLibrary）"
- "Steam ID（SteamID64）"

页面顶部直接是"国服游戏来源"。`WutheringWavesPage.xaml.cs` 在 `OnNavigatedTo` 中改为从 `_settings` 读取 `SteamInstallPath / SteamLibraryPath / SteamId`；"生成 ACF"和"复制启动命令"逻辑改用全局值。

#### 新增页面 `Views/Pages/CustomManifestPage.xaml/.cs`

布局结构（自上而下）：

```
ScrollViewer > StackPanel Padding=24 Spacing=16 MaxWidth=1000
├── InfoBar  (Severity=Warning, "请先在「设置」中配置 Steam 全局参数"; 已配置时 IsOpen=False)
├── Border "游戏标识"
│     ├── AppID                TextBox
│     ├── DepotID              TextBox
│     ├── 显示名称             TextBox
│     ├── 安装目录名           TextBox
│     └── 语言代码             ComboBox (schinese/english/japanese/koreana/tchinese)
├── Border "游戏可执行文件"
│     └── Client EXE   TextBox + [浏览] 按钮
├── Border "BuildID 和 Manifest"
│     ├── [从 SteamDB 自动获取] 按钮
│     └── BuildID / Manifest 两列 TextBox
├── StackPanel "操作"
```

页面级日志沿用鸣潮页模式（`ItemsControl logList` + 内存 `ObservableCollection<string>`），不与鸣潮页日志混合。所有用户操作同时调用 `LogService.Info(...)` 写入磁盘日志。

#### `MainWindow.xaml` 导航项

在"鸣潮"和"敬请期待"之间插入：

## 4. 数据流

```
[用户在设置页填写 Steam 全局配置]
        │
        ▼
   SettingsService.Save(AppSettings)
        │
        ├─→ 鸣潮页 OnNavigatedTo 读取 → 生成 ACF / 启动命令
        │
        └─→ 自定义页 OnNavigatedTo 读取 → InfoBar 状态判定
                                          → 生成 ACF (AcfParameters) / 启动命令

[用户在自定义页填写 + 保存]
        │
        ▼
   AppSettings.CurrentCustomManifest 更新
        │
        ▼
   SettingsService.Save(AppSettings)
```

ACF 文件输出路径：`<SteamLibraryPath>\steamapps\appmanifest_<AppID>.acf`（鸣潮页与自定义页相同规则，仅 AppID 不同）。

## 5. 错误处理

| 场景 | 行为 |
|------|------|
| 自定义页未配置 Steam 全局参数即点"生成 ACF" | 弹 `ContentDialog` 提示，引导跳转设置页 |
| AppID/DepotID 非数字 | TextBox 失焦时 `LogService.Warn` + 红色边框（沿用 WinUI 验证样式） |
| Client EXE 路径不存在 | 生成启动命令前 `File.Exists` 校验，失败弹 `ContentDialog` |
| SteamDB 获取失败 | InfoBar 提示"自动获取失败，请手动填写或访问 steamdb.info/app/<AppID>/depots/" |
| ACF 文件写入失败（权限/路径无效） | `LogService.Error` + ContentDialog 显示异常消息 |
| settings.json 反序列化失败 | 当前已 try-catch 返回新 `AppSettings()`，新字段默认值兼容 |

## 8. 风险评估

| 风险 | 等级 | 应对 |
|------|------|------|
| `SharedDepots` 共享段对非鸣潮游戏的兼容性 | MEDIUM | 沿用现有 `228989/228980`（Steamworks 通用运行时），v2.3 视用户反馈再考虑暴露 |
| 鸣潮页搬迁后字段同步语义变更（原"控件变化即保存"是否保留？） | LOW | 全局字段移到设置页后由设置页负责保存；鸣潮页只读使用，避免双写冲突 |
| `appmanifest_<AppID>.acf` 文件名冲突 | LOW | 文件名自带 AppID，互不覆盖 |
| `SteamDB API` 不同 AppID 的 JSON 结构差异 | LOW | 现有解析逻辑已对 `manifests.public` 同时支持字符串和对象两种格式 |
